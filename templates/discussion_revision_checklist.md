# Discussion Revision Checklist

このチェックリストは、症例報告の考察を校正するときに使用する。
各症例では `manuscripts/{case_id}/submission/10_revision_notes.md` または投稿前チェックリストに結果を転記する。

## 1. 臨床判断の問い

- [ ] 考察全体の中心となる問いを1文で定義した。
- [ ] その問いが本症例の新規性または報告価値と対応している。
- [ ] 考察の結論が、その問いに答える形になっている。

記入欄:

```text
本症例で最も難しかった臨床判断:
```

## 2. 冒頭段落

- [ ] 冒頭段落で本症例の特徴が簡潔に示されている。
- [ ] 冒頭段落で「なぜ報告する価値があるか」が分かる。
- [ ] 症例経過の単なる再掲になっていない。
- [ ] 「稀である」だけで終わっていない。

## 3. 初期治療選択

- [ ] 初期治療選択の妥当性を文献またはガイドラインで確認している。
- [ ] 初期判断の妥当性と、その後の問題を分けて論じている。
- [ ] 次段落への橋渡し文がある。

橋渡し文の例:

```text
したがって、本症例の問題は初期治療選択そのものではなく、その後に制御困難な病態へ移行した時点での判断にあった。
```

## 4. 既報との比較

- [ ] 既報の一般論だけでなく、本症例との共通点を書いている。
- [ ] 本症例が通常経過から逸脱した点を書いている。
- [ ] 頻度や予後の数値は必要最小限にしている。
- [ ] 文献の内容と本文中の主張が直接対応している。

## 5. 治療戦略の説明

- [ ] 処置を時系列に羅列するだけになっていない。
- [ ] 各治療を病態生理または治療意図で説明している。
- [ ] 治療戦略が段階的に整理されている。
- [ ] その治療を選んだ理由と、奏効しなかった理由が区別されている。

治療意図の整理例:

```text
1. 漏出源の尿産生を減らす
2. 産生尿を別経路へ逃がす
3. 残存瘻孔の閉鎖を図る
```

## 6. 奏効しなかった理由

- [ ] 症例情報に基づく範囲で、奏効しなかった理由を慎重に考察している。
- [ ] 推測は「可能性がある」「関与した可能性がある」など慎重な表現にしている。
- [ ] 入力情報にない解剖学的詳細や検査所見を追加していない。
- [ ] 文献で確認していない一般論を断定していない。

## 7. 小児心理社会的負担

- [ ] 小児心理社会的負担を独立した一般論として置いていない。
- [ ] 長期入院、チューブ依存、学業、家族・友人関係、治療終期不明などを治療継続の不利益として統合している。
- [ ] 精神面について、症例情報を超えた診断名や詳細な心理状態を推測していない。
- [ ] 文献はリスクベネフィット判断を補強するために使っている。

## 8. 段落間のつながり

- [ ] 1段落1テーマになっている。
- [ ] 段落冒頭でその段落の役割が分かる。
- [ ] 段落末に次段落への橋渡しがある。
- [ ] 「文献では」で始まる段落が連続していない。
- [ ] 「本症例は〜を示唆する」を多用していない。

## 9. 結論と限界

- [ ] 最後に本症例から得られる臨床的示唆が明確に書かれている。
- [ ] 限界は短く、単一症例であることや一般化困難性を過不足なく記載している。
- [ ] 限界で終わる場合でも、臨床的示唆が直前または直後に明確である。

## 10. AI感の除去

- [ ] 文献の羅列ではなく、臨床判断の流れになっている。
- [ ] 同じ表現や同じ段落構造が繰り返されていない。
- [ ] 各段落が「本症例で何が重要だったか」に戻っている。
- [ ] 読者が、筆者がどこで悩み、どう判断したかを追える。

## 修正メモ

```text
修正した点:

残る課題:

投稿前に人が確認すべき点:
```
